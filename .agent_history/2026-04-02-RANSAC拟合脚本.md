# Agent History

## 2026-04-02 - RANSAC拟合脚本开发

### 模块变更总结
- 新增 `nihe.py`：使用 RANSAC 对 X/Y/Z 轴数据进行指数拟合并可视化
- 新增 `nihe_line.py`：使用 RANSAC 对数据进行二次多项式拟合
- 两个脚本均读取 `sub_points_X轴.pkl`、`sub_points_Y轴.pkl`、`sub_points_Z轴.pkl` 数据文件

### 对话过程
1. 用户要求将 `nihe.py` 从 `curve_fit` 改为 RANSAC 拟合，并添加绘图功能
2. 采用对数变换 `log(x) = log(A) + B*y` 将指数模型线性化，配合 `RANSACRegressor` 实现鲁棒拟合
3. 用户要求限制 y 轴范围为 0-1
4. 用户要求创建 `nihe_line.py`，使用 `Ax² + Bx + C` 二次拟合
5. 用户要求 `nihe.py` 读取 XYZ 三轴数据，分别绘制在三个子图中
6. 用户要求颜色分别使用 RGB（红/绿/蓝），离群点使用叉号标记
7. 用户要求保存图像为 PNG，字体调大

### 关键决策
- 指数拟合通过对数变换转化为线性问题，再用 RANSAC 做鲁棒回归
- 二次拟合使用 `PolynomialFeatures(degree=2)` + `RANSACRegressor` pipeline
- 三轴数据分别绘制在三个子图中（1行3列），而非合并到同一坐标系
- 离群点使用黑色叉号（`marker="x"`）与内点圆点区分
- 图像保存为 `nihe_exp_ransac.png`，300dpi

### 技术细节
- 字体设为宋体（SimSun/Songti SC/STSong），全局字号 12
- 内点/离群点通过 `ransac.inlier_mask_` 区分
- 数据中 x ≤ 0 的点被过滤（对数变换要求正值）
- 图表尺寸 `figsize=(12, 4)`，y 轴范围 `(-0.05, 1.05)`
